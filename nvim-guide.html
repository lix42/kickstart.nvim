▍Setup — just open a folder

No project setup needed. Open Neovim in your project root:

cd ~/your-project
nvim .          # or `nvim somefile.lua`

What happens automatically:

  • vim.pack (Neovim's built-in plugin manager, 0.12+) loads plugins and self-installs them on first run — no external manager. Run :lua vim.pack.update() to update; versions are pinned in nvim-pack-lock.json.
  • LSP attaches per-file via Mason. Configured servers: lua_ls, ts_ls (TS/JS + React JSX/TSX), rust_analyzer, cssls, tailwindcss, html, omnisharp (C#), pyright (Python) — plus stylua for formatting. Add more in the servers = {} table in init.lua; :Mason shows install status.
  • Treesitter highlights & indents (parsers auto-install when you open a new filetype).
  • Auto-save on buffer-leave / focus-lost, and files auto-reload if changed on disk. You rarely type :w.
  • neo-tree file explorer — press \ to reveal the current file in a sidebar (and again to close it).

▍Markdown (,m prefix)

Two complementary previews: render inside the editor, or open a live preview in the browser.

KeyAction
,mrRender markdown in-editor (render-markdown.nvim — headings, tables, code blocks, checkboxes). Toggles; hides itself in insert mode so you edit raw source.
,mpPreview in the browser (markdown-preview.nvim — live-syncing). Toggles the preview tab.
First use of ,mp needs the preview binary; it downloads automatically when the plugin installs. If it ever errors, run :call mkdp#util#install() once.

▍Typical editing loop

  1. nvim . in the repo.
  2. ,sf / Ctrl+p to open a file, or ,sg to grep for something.
  3. grd to jump to a definition, Ctrl+t to come back; grr for references.
  4. Edit (auto-saves on buffer switch; ,f to format).
  5. grn to rename, gra for quick-fixes.
  6. ,gd to review your diff before committing.
